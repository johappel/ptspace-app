# UI-Sprache

Die Oberfläche spricht Lehrkräfte an, nicht Entwickler:innen oder Administrator:innen.

## Verwenden

- Planungsräume
- Gemeinsam nachdenken
- Denkstand
- Offene Entscheidungen
- Nächste Schritte
- Entwürfe
- Für den Unterricht bereit
- Geschützter Planungsraum
- prüfen
- weiterdenken
- als Entwurf vorbereiten
- für den Unterricht freigeben

## Nicht im Lehrer:innen-Modus verwenden

- opencode
- Harness
- Workspace
- Git
- Commit
- Branch
- YAML
- Shell
- Terminal
- Service Request
- Render-Artefakt
- API-Key
- Token
- Provider-Prompt

## Typografie

Sichtbare deutsche UI-Texte verwenden echte Umlaute. Ersatzschreibungen wie `ae`, `oe` oder `ue` sind in der UI Typografiefehler.

## Technische Begriffe

Technische Begriffe dürfen intern in Code, Tests und Architektur-Dokumentation vorkommen. Sobald ein Text in der Lehrer:innen-Oberfläche, in einer API-Fehlermeldung oder in einem Export sichtbar werden kann, muss er pädagogisch übersetzt werden.

## Datenschutz-Hinweis

Die UI soll zur Datenminimierung ermutigen, ohne die Lehrkraft zu belehren:

> Für die Planung reichen Beschreibungen ohne Namen einzelner Schüler:innen.
